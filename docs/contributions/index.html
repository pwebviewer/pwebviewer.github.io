How to get involved

01

Read and comment on the spec. Open issues or pull requests on portableweb/spec. The format is still in draft — early feedback shapes it most.

02

Build something with it. Create a .pweb bundle and share it. Real-world usage drives the spec far more than theory.

03

Contribute code. The CLI, viewer, and website repos all welcome PRs. Check the open issues on github.com/portableweb for good starting points.

04

Spread the word. Tell anyone who works with documents, archival, AI-generated content, or interactive media. The format only matters if people know it exists.
